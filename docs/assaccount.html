AWS Services

Overview
Get familiar with AWS services

Description

This assingment is going to cover some of the important Amazon Web Services and get you started with elastic cloud and server-less scripting. You will be need to fix errors related to Node application and find your way around an actual Linux server.

Instructions

  • Watch the lectures about Cloud
  • Create an AWS account ( You can cancel your account after class, you would not be charged for free tier services)
  • Go to "Compute" and Launch an EC2 Free Tier instance.
  • Select a Free Tier Ubuntu Server 18.04 Amazon Linux Image
  • Select t2.micro Free Tier
  • Click to Configure Instance Details
  • Configure Instance Details
  • Change "Auto-assign Public IP" to enable
  • Click on add storage and leave the default values
  • Click on Review and Launch
  • Click on Launch
  • Change option to "Create new key pair"
  • Type the name of the keypair (anything)
  • Click on "Download" (if you loose these you can access to that server and need to start over)
  • Click on Launch Instance
  • Click on Launch View Instances
  • Once the EC2 is changed from "Pending" to "Running" continue
  • Find the IPv4 Public IP address and take a screen-shot
  • open terminal and access to this ec2

>> ssh -i youkey.pem ec2-user@ipnumber

You will get an error about the wrong permission. Look at the screen-shots to fix this issue.
You will than be able to login to your ec2 after that execute the commands and take scree-shots.

  • run command "whoami" and take a screen-shot ( this should be ec2-user)
  • run command "pwd" and take a screen-shot (this should be /home/ec2-user)
  • run command "sudo yum update type y"
  • Follow this page to setup node https://docs.aws.amazon.com/sdk-for-javascript/v2/developer-guide/setting-up-node-on-ec2-instance.html
  • Screen-shot the output of the "node -e "console.log('Running Node.js ' + process.version)"
Let's open ports to our Node Server
  • Click on "EC2" (Home page)
  • Click on "Security Groups"
  • Select "launch-wizard-1"
  • Click on "Actions"
  • Click on "Inbound"
  • Click on "Add Rule" and enter Port Range (8080)
  • Click on "Save"
Go back to your SSH window in your server
